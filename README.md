## What I Have Done

This project implements a hospital system with one TCP front-end server and three UDP back-end servers. The client authenticates through the Hospital Server, and the Hospital Server coordinates with the Authentication Server, Appointment Server, and Prescription Server to process patient and doctor requests.

Implemented features include:

- User authentication with SHA-256 hashed username and password
- Role-based access for patients and doctors
- Doctor lookup and doctor availability lookup
- Appointment scheduling, appointment viewing, and appointment cancellation
- Prescription creation by doctors
- Prescription viewing for both doctors and patients
- Persistent storage using the provided text files

## Code Files

- `client.py`: Runs the interactive client program for patients and doctors and sends TCP requests to the Hospital Server.
- `hospital_server.py`: Runs the central server, accepts TCP client connections, authenticates users, and forwards requests to the back-end servers over UDP.
- `authentication_server.py`: Verifies hashed username/password pairs against `users.txt`.
- `appointment_server.py`: Stores and manages doctor schedules and patient appointments using `appointments.txt`.
- `prescription_server.py`: Stores and returns prescription information using `prescriptions.txt`.
- `common.py`: Contains shared constants, file paths, hashing functions, file parsing helpers, and JSON socket helper functions.
- `Makefile`: Provides commands to syntax-check the project and start each server/client program.
- `README.md`: Documents the project, file roles, message formats, assumptions, and submission details.

## Data Files

- `users.txt`: Stores hashed usernames and hashed passwords for authentication.
- `hospital.txt`: Stores doctor name to hash mappings and illness to treatment mappings.
- `appointments.txt`: Stores each doctor's appointment slots and any scheduled patient/illness data.
- `prescriptions.txt`: Stores prescription records generated by doctors.

## Message Formats

All inter-process messages are JSON objects encoded in UTF-8.

TCP messages between Client and Hospital Server:

- Authentication request:
  `{"action":"authenticate","username_hash":"<sha256>","password_hash":"<sha256>"}`
- Patient command request:
  `{"action":"command","role":"patient","command":"<command>","username_hash":"<sha256>", ...}`
- Doctor command request:
  `{"action":"command","role":"doctor","command":"<command>","username":"<doctor_name>","username_hash":"<sha256>", ...}`
- TCP responses:
  JSON objects containing at least a `status` field and additional command-specific fields.

UDP messages between Hospital Server and Authentication Server:

- Request:
  `{"action":"authenticate","username_hash":"<sha256>","password_hash":"<sha256>"}`
- Response:
  `{"status":"ok","authenticated":true|false}`

UDP messages between Hospital Server and Appointment Server:

- Lookup doctors:
  `{"action":"lookup_doctors"}`
- Lookup doctor availability:
  `{"action":"lookup_availability","doctor":"<doctor_name>"}`
- Schedule appointment:
  `{"action":"schedule","doctor":"<doctor_name>","time":"HH:MM","patient_hash":"<sha256>","illness":"<illness>"}`
- Cancel appointment:
  `{"action":"cancel","patient_hash":"<sha256>"}`
- View patient appointment:
  `{"action":"view_patient_appointment","patient_hash":"<sha256>"}`
- View doctor appointments:
  `{"action":"view_doctor_appointments","doctor":"<doctor_name>"}`
- Get patient illness for prescribing:
  `{"action":"get_patient_illness","patient_hash":"<sha256>","doctor":"<doctor_name>"}`

UDP messages between Hospital Server and Prescription Server:

- Add prescription:
  `{"action":"add_prescription","doctor":"<doctor_name>","patient_hash":"<sha256>","treatment":"<treatment>","frequency":"<frequency>"}`
- View prescriptions:
  `{"action":"view_prescriptions","patient_hash":"<sha256>"}`

## How to Build and Run

Use the provided `Makefile`.

Syntax check:

```bash
make check
```

Start the programs in separate terminals in this order:

```bash
make run_hospital
make run_authentication
make run_appointment
make run_prescription
make run_client USER=<username> PASS=<password>
```

You can start additional clients with another `make run_client` command in a new terminal.

## Idiosyncrasies / Known Limitations

- The project uses `127.0.0.1`, so all processes are expected to run on the same machine.
- The program assumes the input files exist in the same directory as the Python files.
- Doctor names and usernames are expected to match the mappings provided in the data files.
- The Appointment Server initializes a default schedule only if `appointments.txt` is empty.
- If a server is not running when another process sends a request, that request will fail.
